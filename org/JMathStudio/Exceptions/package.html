This package contain classes defining various Java exceptions used in the toolkit.

Exceptions

An exception is thrown by a method to indicate a specified illegal state. Following Exceptions are defined in the toolkit :

  • BugEncounter Exception
    • This Exception is thrown by a method if the code within the method encounter any Illegal state arising due to a bug in the programming or an algorithm failure.
  • IllegalArgument Exception
    • This Exception is thrown by a method if the arguments passed on to the method are illegal or not within the constraint.
  • DimensionMisMatch Exception
    • This Exception is thrown by a method if it encounters any dimension mismatch between the objects ie. Data Structures, images etc.
  • DivideByZero Exception
    • This Exception is thrown by a method if any divide by zero condition arises.
  • IllegalCellFormat Exception
    • This Exception is thrown by a method if the Cell data structures do not adhere to specified format.
  • ObjectNotFound Exception
    • This Exception is thrown by a method if the object in question ie. file, external image, object etc is not available.
  • ObjectOverWrite Exception
    • This Exception is thrown by a method if the method is forced to over write on an existing object ie. file, image etc.
  • PixelImageFormat Exception
    • This Exception is thrown by a method if it encounters an illegal PixelImage format.
  • InvalidPixelFormatException
    • This Exception is thrown by a method if the pixel parameter and/or argument is invalid for the given Pixel type.
  • UnSupportedImageFormat Exception
    • This Exception is thrown by a method if it encounters an undefined or unsupported image format or type.
